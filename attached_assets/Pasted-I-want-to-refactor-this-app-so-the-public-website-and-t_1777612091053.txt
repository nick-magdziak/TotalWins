I want to refactor this app so the public website and the live sports score updater are separated into two deployment targets.

Current problem:
The web server currently both serves the app and runs background sports sync jobs/intervals. I want to move all score syncing/background worker logic out of the web server so I can deploy the website as Autoscale, while keeping a small Reserved VM worker for near-instant live score updates.

Please make the following changes:

1. Refactor server/index.ts so it is website/API only
- Remove startup sports sync calls from server/index.ts.
- Remove the recurring 2-minute sports sync interval from server/index.ts.
- server/index.ts should only initialize the Express app, register routes, serve the frontend, and start the HTTP server.
- Do not change the normal website/API behavior.

2. Remove background score interval from server/routes.ts
- Find the 30-minute interval that calls something like:
  sportsApi.syncGamesForWeek(3, "2025")
- Remove it from routes.ts.
- routes.ts should register API routes only, not run background worker loops.

3. Create a dedicated live score worker
Create a new file:

scripts/live-score-worker.ts

This worker should:
- Import the same sports services currently used by the web server.
- Run the same sports sync operations that are currently being triggered in server/index.ts, including:
  - updateMLBStandings()
  - updateNFLStandings()
  - updateNBAStandings()
  - sportsApi.syncMLBGames()
  - sportsApi.syncNBAGames()
  - sportsApi.syncCurrentNFLGames()
  - sportsApi.syncNextWeekNFLGames()
  - sportsApi.syncTeamStandingsFromESPN()
- Run once immediately on startup.
- Then loop continuously.
- During live-game periods or when live games are detected, sync every 1–2 minutes.
- Outside active game windows, sync less often, such as every 30–60 minutes.
- Add clear logging so I can see when sync starts, when it finishes, and any errors.
- Catch errors so one failed sync does not crash the worker permanently.
- The worker should write to the same Postgres/Neon database using DATABASE_URL.

If it is difficult to detect live games perfectly, implement a simple safe version first:
- Sync every 2 minutes for now.
- Add TODO comments showing where live-game-aware throttling can be added later.

Add scripts similar to:

Find the draft pick creation route. It currently builds pickData from req.body, userId, and leagueId.
Change it so the draft pick sport is always set from the league’s sport:
